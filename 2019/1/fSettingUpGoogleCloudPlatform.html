Setting Up Google Cloud Platform

John Manalastas | Sat 05 October 2019

Why Setup a GCP¶

For most aspiring data scientists on a budget, you will quickly realized that your current rig is not up to par with the projects you might want to tackle. I expirenced this with my personal laptop, where both memory or computing power are not able to do the data crunch.

A solution to this problem is to create virtual machine (vm). In my case, I used the Google Cloud Platform as opposed to AWS EC2. The GCP is developed enough that it is competitive against AWS.

This tutorial will include the following:

  • Create an instance on Google Computer Engine (GCE)
  • Customize both firewall and ip address for easy use
  • Install Jupyter notebook and popular libraries related to data science
  • Run Jupyter notebook accessible online
  • Configure Kaggle api to download data

Advantages of cloud computing over local machine¶

There are two main advantages when working with cloud:

  • First, it is scalable. You are able to tailor your machine's RAM, CPU, GPU and memory for current use and add more power to each as needed.
  • Second, it is easily shared. Yes, we have github for that, but the cloud can be instantly accessed from any machine at anytime, as long as the vm instance is open.

Getting Started¶

Create an Account and Project:¶

To use the GCP you need to create a Google account. At the time of this writing a new account with a new debit/credit card is given a 12 months or 300 usd free trial of the platform, whichever runs out first. The 300 usd credit is enough to get you started and if you need more time with the vm, you are always free to register another card.

To create your GCP account go to link(http://cloud.google.com) and click on the "try it free button. You'll be asked to login with your google account and billing info. Once that is done go to link(http://console.cloud.googl.com)

Step 3: Create a VM instance¶

Click on the three lines on the upper left corner, then on the COMPUTER option, click on 'Compute Engine'

Now click on 'Create new instance.' Name your instance, select zone as ‘ us-west1-b’. Choose your ‘machine type.’

Select your boot disk as "Ubuton 16.04 LTS." Under the firewall options select both 'http' and 'https.' Then, choose the disk tab and untick ‘ Delete boot disk when instance is deleted’.

If you click on ‘customize’, you will be able to find options for using GPUs. You can choose between 4 different NVIDIA Tesla GPUs.
