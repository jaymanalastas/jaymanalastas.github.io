Deep Dive into Seaborn¶

Overview:¶

  • Seaborn is a popular data visualization library for Python
  • It is a combination of aesthetic appeal, technical insights and ease of use.
  • Sample sns plots, graphs and customization tools available through seaborn

Introduction¶

Seaborn is built based on $matplotlib$ library. It is used to create well-designed visualization with nice aesthetic quality and provides meaningful insights. As a Seaborn user, you can customize the color, so it stands out, define boundaries to see data more clearly and add different parameters to gain more insight with a messy data.

As a data scientist, our job is to provide explanations to our stakeholders/clients. Often, these people rely on us and our data to make snap decisions. Having visual representation of our findings and insight coupled with our presentation will go a long way in our careers.

Although there are other Python libraries for visualization, Seaborn is a clear favorite of mine due to ease of use and technical insight it provides.

def resumetable(df):
    print(f"Dataset Shape: {df.shape}")
    summary = pd.DataFrame(df.dtypes,columns=['dtypes'])
    summary = summary.reset_index()
    summary['Name'] = summary['index']
    summary = summary [['Name', 'dtypes']]
    summary['Missing'] = df.isnull().sum().values 
    summary['Uniques'] = df.nunique().values
    summary['First Value'] = df.loc[0].values
    summary['Second Value'] = df.loc[1].values
    summary['Third Value'] = df.loc[2].values
    
    for name in summary['Name'].value_counts().index:
        summary.loc[summary['Name'] == name,
                    'Entropy'] = round(stats.entropy(df[name].value_counts(normalize=True),
                                                     base=2),2)
    return summary

Distribution plot¶

A bar chart is a common example in visualizing the distribution of a single feature in a dataset. Each instance a data is expressed it is mapped in a 2-d grid in between a bin and the more we get the higher the bin gets.

To initiate this we use $distplot()$ function in the Seaborn library.

The syntax goes as follow:

sns.distplot(tips['total_bill'])

Joint plot¶

Joint plot is a useful graph that creates a visual relationship between two variables. Each point is created using the columns as a map to point that particular data in a 2-d graph.

The parameters we use are $x$, $y$, and $data$. These will populate the x-axis, y-axis and the data we are using. In this example we will explore the relationship between bill amount and tips.

Based on experience, we should expect a positive correlation between tip and bill amount.

we can use the following to change the kind of plots we're using.

$sns.jointplot(x="total_bill", y="tip", data=tips, kind = ('reg','hex''kde')$

reg = regression line through a scatter plot. hex = instead of a plot it'll use hexagon with darker color to indicate frequency kde = shows density of the plots

In our example, we will use $reg$ as kind and it'll create a scatterplot with a regression line on top of it.

Bar plot¶

A Bar plot is a general plot that allows you to aggregate the categorical data based off some function; the default function is the mean.

function is $sns.barplot(x='sex ',y='total_bill ',data=tips, estimator='statistical function ')$

Note - we can add more statistical funciton to the $estimator$ in sns.barplot.

By default, it is calculating the mean, but we can create our own statistical function within the $estimator$ bin.

Here the categorical column is "sex." Based on the graph below, the average bill for males is slightly higher than the average bill for females.

Box and Violent plot¶

Both these pots are used to show the distribution of categorical data. A box plot is AKA box and whisker plot. The graph shows the distribution of quantitative data in a way that hopefully facilitates comparisons between variables.

$sns.boxplot(x='day',y='total_bill',data=tips)$

To make it more interesting we will add $hue='smoker'$ and it will look like: $sns.boxplot(x='day',y='total_bill',data=tips, hue='smoker')$

Now we have a graph that splits our data by day and smoker status. We're able to see the behavior of the column 'total_bill' in terms of 'days' and 'smoker' features.

This is where the power of Seaborn shines. We're able to quickly add parameters to our graphs quickly and smoothly.¶

Strip & Swarm plot¶

Is a scatterplot where one variable is categorical.

Example: $sns.stripplot(x='day', y='total_bill', data=tips, jitter= True)$

$jitter$ adds random noise to separate some of the stacked points. This enables us to see the thicker points and provides us a clearer version of a strip plot.

Similar above, we will add $hue$ and $split(aka- dodge)$ arguments to get a better feel of our data.

Swarm plot¶

Swarm plots is very similar to a strip plot, but the points do not overlap. This gives a better representation of the distribution of values. A good way to think about it is a combination of both scatter and violent plots.

The drawbacks of a swarm plot are if the data is too big, the points might go too wide and it also takes a lot of computation to begin to arrange them. So, it might be wise to use the swarm plot only on manageable datasets.

Heat map/Cluster map¶

Heat map grid is interesting, because before we can use the function, we need to dress the data into a matrix form. In this instance, we will use the flights data that is included in Seaborn.

We will use the column 'month' as the index, column 'year' as column and use 'passenger' as values.

$fp = flights.pivot_table(index='month', columns='year', values='passengers')$

We will then use the $sns.heatmap(fp)$ to run the heat map function.

The cluster map creates a hierarchal clustering to produce a clustered version of the heatmap.

The function $sns.clustermap(fp)$ basically tires to cluster columns and rows together based off their similarities. This then gives us a bird’s eye view of similarities between the parameters we’ve included.

In [31]:
fp = flights.pivot_table(index='month',
                         columns='year', values='passengers')


sns.heatmap(fp, cmap='coolwarm')


sns.clustermap(fp, cmap='coolwarm', standard_scale=1)
Out[31]:
<seaborn.matrix.ClusterGrid at 0x7f0f2f480710>

Based on the above, the months of Nov, Jan, and Feb are the winter months and usually have lower count of passengers.

In contrast, July and Aug are categorized as higher passenger months.

Pair Grid¶

As discussed before, pair plots are automated joint plots for the entire data set. A Pair Grid is a little more customizable. We are able to dictate which kind of plots shows in these 3 regions:

  • $g.map - diag$
  • $g.map - upper$
  • $g.map - lower$

We usually use only pair plot because it is easier, but if you want to get fancy then a $PairGrid$ plot is the way to go.

In [49]:
g = sns.PairGrid(iris)
g.map_diag(sns.distplot)
g.map_upper(plt.scatter, color='red')
g.map_lower(sns.kdeplot, cmap = 'coolwarm')
Out[49]:

Facet Grid¶

Is a further customization of our dataset. for the grid to work we must call in 3 parameters. $snsFacetGrid(data='',col='',row='')$

This will give us an empty grid and we can call up any numerical columns we want to show its behavior when taking into account the col or row we passed in our $FacetGrid$.

Here we will pass the $total_bill$ column to show its behavior towards the two categorical columns.

$g.map(sns.distplot, 'total_bill')$

In addition, we can pass in other types of plots in the $Facet Grid$. For example, we'll also use a $scatter plot$ to map out the behavior of both $tips$ and $total_bill$ in relationship with $time$ and $smoker$ categories

Regression Lines¶

Below is the $sns.lmplot$, it is similar to the joint plot but we are able to customize this further. We use the following:

  • $markers$ are used to distinguish the hues we passed in
  • $row/ col$ Variables that define subsets of the data, which will be drawn on separate facets in the grid.
  • $hue$ color codes the data value within the grid to the column
  • $aspect$ Aspect ratio of each facet, so that aspect * height gives the width of each facet in inches.
